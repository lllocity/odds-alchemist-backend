# プロジェクト概要: Odds Alchemist
Yahoo!スポナビ競馬のオッズ情報を定期的にスクレイピングし、Google Sheetsに保存。その後、オッズの異常（支持率の急増、単複オッズの順位乖離など）を検知してフロントエンドに通知するシステム。

## 技術スタック
- **構成**: モノレポ構成
- **バックエンド (`/backend`)**: Java 17+, Spring Boot, Jsoup (スクレイピング), Google Sheets API
- **フロントエンド (`/frontend`)**: Next.js (App Router), React, Tailwind CSS
- **データストア**: Google Sheets

## 現在のシステム状態
- FEからBEの `/api/odds/fetch` エンドポイントを叩き、対象レースURLを送信する疎通は完了している。
- BEは受け取ったURL（スポナビ）のHTMLをJsoupで取得し、テキストパターン解析によって「馬番」「馬名」「単勝」「複勝」を抽出している。
- 抽出したデータはGoogle Sheets API経由でシートに書き込まれている。

## コーディングの基本方針
- エラーハンドリングとログ出力（SLF4J）を適切に行うこと。
- クラスの責務を明確にし、肥大化を防ぐこと。
- 破壊的変更を行う場合は、既存のテストコードも合わせて修正すること。
